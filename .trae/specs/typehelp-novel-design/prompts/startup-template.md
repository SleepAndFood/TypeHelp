# 项目启动模板（Director 输入）

> 用户填好此模板后，发送给 Director Agent，Director 会按此立项并产出 `charter.md`。

---

## 用户填写

```yaml
# 必填字段
题材: <从以下选一>
  - 嘉利别墅型（暴风雪山庄 / 古堡）
  - 办公室密室
  - 太空船 / 潜艇 / 封闭环境
  - 民国药铺 / 民国侦探
  - 现代都市
  - 校园 / 医院
  - 其他（请说明）

风格: <从以下选一>
  - 古典本格（公平博弈、读者与侦探平等）
  - 新本格（叙述性诡计 / meta）
  - 社会派（动机重于诡计）
  - 心理悬疑
  - meta 恐怖

目标时长: <30 / 60 / 120 分钟>
目标玩家: <推理老手 / 新手 / 混合>
难度: <1-5 星，5 星最难>

# 可选字段
特殊约束:
  - <如：避免暴力 / 必须含女性视角 / 单机离线 / 不可超自然 / 时代限制>
  - <多条用 `-` 开头>

# 灵感来源（可选）
参考作品: <如：游戏名 / 小说名 / 剧本名>
参考诡计: <如：时间诡计 / 身份诡计 / 信息差诡计>
meta 元素: <如：调查员视角 / 嵌套循环 / 旁观者>

# 偏好（可选）
结局数: <单结局 / 多结局（请说明）>
是否有音频: <是 / 否>
是否需要画外音: <是 / 否>
```

---

## 示例

```yaml
题材: 太空船 / 封闭环境
风格: 新本格
目标时长: 60 分钟
目标玩家: 推理新手到中级
难度: 3
特殊约束:
  - 单机离线
  - 不可超自然
参考作品: 死亡搁浅 / 她和她的她
参考诡计: 信息差诡计 + 物证诡计
meta 元素: 调查员视角
结局数: 单结局
```

---

## Director 收到后的工作流

1. 解析 6 个必填字段
2. 选定核心诡计（从六大类中选：时间 / 身份 / 物证 / 信息差 / 视角 / 心理）
3. 估算文件数（默认 95-100，演示规模可缩）
4. 设定幕数（默认 4 幕）
5. 评估风险（至少 3 项）
6. 撰写 `charter.md`
7. 给 Truth Designer 写 handoff

---

## Director 的 handoff（Truth Designer 输入）

Director 完成后会写类似：

```markdown
# Handoff: Director → Truth Designer

## 接收
- charter.md：8 字段齐全
- 核心诡计：信息差 + 物证
- 风险：6 项已记录

## 期望的 truth.md
- 时间线：发射后第 89 天（约 6 月）
- 地点：太空船"曙光号"6 个区域
- 人物：6 名船员 + 1 名地球控制（K）+ 1 名 meta（@）
- 诡计：医生篡改 PLC 氧气参数，致指挥官睡眠中窒息
- 物理可行性：必须 100% 通过

## 约束
- 不可超出 TypeHelp C2 约束（无询问 NPC）
- 真相层一旦冻结，修改需走 RFC

## Deadline
无，但下游 Inference Architect 在等。
```

Truth Designer 收到后开始撰写 `truth.md`。
