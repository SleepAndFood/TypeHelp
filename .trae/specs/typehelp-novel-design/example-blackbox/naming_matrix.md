## 4. presence list 编码规则

**严格规则**：
- ID 列表必须按数字升序排列（除 K、@ 外）
- 不在场的角色**禁止**出现在 ID 列表中
- 同事件不同视角文件 ID 列表必须**完全一致**（who-was-there 不可变）

**示例**：
- 04-EN-1-2-4 表示：04 时间 / EN 引擎舱 / 1, 2, 4 在场
- 22-AT-1-2-3-4-5-6 表示：22 时间 / AT 中庭 / 全员在场
