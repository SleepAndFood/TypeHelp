## 3. 互引图自检

### 3.1 可达性

从 `00-readme` 出发：
```
00-readme → 01-CR-1 → 02-BR → 03-ME / 04-EN
                       ↓
              05-CA → 09-CA → 12-CO → 14-LA → 18-LA → 21-AT → 22-AT → 25-WI
```

所有公开文件可达 ✓

### 3.2 孤立节点

无 ✓

### 3.3 环

无（线性时间推进）✓

### 3.4 隐藏文件触发

- **00-dream**：必须已知文件名（玩家通过 `00-references` 中提到"一些文件被排除在文件列表外"得知）
- **00-final-note**：玩家收集到 `$cache_max - 2` 个文件后，`$is_complete = true`，inbox 显示 2 条消息，读完 message2 后解锁
- **25-WI-2**：必须先读完 22-AT（其 tag 指向 25-WI-2）

### 3.5 教程引用链

- `00-references` → `01-CR-1`（教导命名规则）
- `02-BR-1-2-3-4-5-6` 中 [@] 提示 `name` 命令 → `$seen_name = true`
- `03-ME-1-2` 中 [@] 提示 `note` 命令 → `$seen_note = true`
- `09-CA-1-2-3-4-5-6` 中 [@] 提示 `find` 命令 → `$seen_find = true`
- `14-LA-4` 中 [@] 提示 `title` 命令 → `$seen_title = true`
- `22-AT` 中 [@] 提示 `act` 命令 → `$seen_act = true`
- `00-final-note` 中 [@] 提示 `hangman` 命令 → `$seen_hangman = true`

## 4. 引用计数

每个文件的 `tags` 数量决定其在 `list` 中显示的灰色数字（参考 TypeHelp 源码 pid 1 list 段 `tag_count -= 1 if $cache.includes(_tags[_j])`）。

例如 `02-BR-1-2-3-4-5-6` 的 tags 是 `03-ME-1-2 04-EN-1-2-4`，若两者都未读，则显示数字 `(2)`；读完一个后变为 `(1)`。
