# Type Help

> 一款基于 SugarCube / Twine 引擎的纯文本交互式推理游戏。

**原作者**：William Rous

**译者**：Akita23508

**引擎**：SugarCube v2.36.1

**格式**：HTML（单文件，浏览器直接打开即可运行）

---

## 游戏简介

你是一名调查员，面前是一台从已故同僚处回收的旧计算机。屏幕上残留着从多具尸体上提取的音频转录文件——它们记录了 **1936 年** 发生在 **嘉利别墅（Galley Villa）** 的一场连环死亡事件。

你的任务：阅读这些文件，拼凑出真相。

---

## 如何运行

1. 下载 `TypeHelp.html` 文件
2. 用任意现代浏览器（Chrome / Firefox / Edge / Safari）直接打开
3. 在文本框中输入命令，按回车执行

无需安装任何依赖，无需联网。

---

## 操作说明

### 基础命令

| 命令 | 用法 | 说明 |
|------|------|------|
| `help` | `help` | 查看帮助信息与可用命令列表 |
| `list` | `list` 或 `list 2` | 查看已收集的文件列表，可按幕（1-4）分页 |
| `back` | `back` | 返回上一页 |
| `save` | `save` | 打开存档界面 |

### 可解锁命令

以下命令在游戏过程中通过叙述者注释逐步解锁，无需刻意寻找——读到对应场景时会自动获得提示。

| 命令 | 用法 | 说明 |
|------|------|------|
| `name` | `name 1 约翰` 或 `name # 你的名字` | 给编号人物取自定义昵称，方便记忆 |
| `note` | `note 内容` | 记录笔记（最多 300 字符），帮你整理线索 |
| `find` | `find 关键词` | 在已发现的文件中全文搜索关键词 |
| `title` | `title 03 标题文字` | 给时间段取自定义标题 |
| `act` | `act 1 标题文字` | 给幕取自定义标题 |
| `hangman` | `hangman` 或 `hangman a` | 猜字母小游戏（后期解锁） |

### 打开文件

直接输入文件名即可打开对应的故事场景文件，例如：

```
01-QU-1-11
```

首次打开的文件会自动收录至文件列表。随着游戏推进你会逐渐理解这套命名规则。

## 游戏机制

- **文件发现系统**：主动输入文件名探索，初始邮件会提供几个起始文件名（其中有一个是损坏的，需要你根据命名规则推断完整文件名）
- **参考数引导**：文件列表中每个文件前的灰色数字表示还有多少未发现的关联文件，数字越大越值得优先探索
- **叙述者注释**：故事中以 `[@]` 标记的旁白会提供线索提示、操作教程和情感评论
- **多视角交叉验证**：同一事件在不同角色的文件中被多次描述，细节可能互补也可能矛盾——对比阅读是推理的关键
- **颜色编码**：不同幕的链接使用不同颜色，帮助你感知叙事进度

## 故事结构

故事分为四幕，共 26 个时间段（01-26），另有若干辅助文件（00-开头）。

| 幕 | 时间码 | 默认标题 |
|----|--------|----------|
| 第一幕 | 01-06 | IGNORANCE（无知） |
| 第二幕 | 07-13 | CONFUSION（困惑） |
| 第三幕 | 14-20 | TERROR（恐怖） |
| 第四幕 | 21-26 | THE RECKONING（清算） |

---

## 注意事项

- **不要急于求成**：耐心阅读每个文件，注意细节
- **善用 `note` 命令**：信息量大，好记性不如烂笔头
- **善用 `find` 命令**：当你需要回溯某个关键词时
- **对比不同视角**：同一事件在不同文件中的描述可能存在关键差异
- **注意叙述者注释**：`[@]` 标记的内容经常包含重要提示
- **文件名本身是线索**：命名规则编码了时间、地点和人物信息

---

## 通关条件

收集足够多的文件后，系统消息会提供最终文件的线索。阅读最终文件后，游戏自动进入结局。

### 核心设定

- 嘉利别墅受到某种 **"遗忘诅咒"**：当人们开始忘记身边已死之人时，诅咒就会蔓延，导致更多人死亡
- 诅咒的根源与嘉利家族的往事有关——奥斯瓦尔德的姐姐埃莉诺童年死于阁楼，未举行葬礼，家人拒绝承认她的死亡
- 教堂是解开诅咒的关键地点

### 关键剧情节点

- **第一幕**：宾客陆续抵达，鲁伯特的尸体在书房被发现
- **第二幕**：管家索顿死亡，晚餐时矛盾升级，伊芙死亡，众人互相猜疑；小艾通过暗门偷走托尼的枪
- **第三幕**：记忆丧失加速，安妮写名单对抗遗忘，钢琴自动演奏，小艾用枪控制局面；安妮在教堂试图用枪自杀以打破诅咒但失败
- **第四幕**：仅剩约翰存活，与凯瑟琳通电话得知别墅已无人居住多年，独自在阁楼等待终结

### 结局

- 调查员在 `00-final-note` 中揭露：他通过尸体音频技术还原了整个事件，意识到自己就是事件链条的终结者——他选择亲自前往嘉利别墅成为最后一名受害者
- 游戏以 hangman ASCII art 和 "THE END" 收束

### 隐藏内容

- `00-dream`：一首双语诗歌（"Because of You"），被排除在文件列表之外，但知道文件名可直接输入打开
- `00-final-note`：调查员遗书，收集足够文件后通过消息系统解锁
- `25-WI-K`：最后一个场景，同样被排除在常规列表之外
- 消息系统中的 YouTube 链接是一个 meta 元素

### 叙述者的真实身份

叙述者（[@]）就是调查员本人。他的"编辑/再编"标注暗示他在反复修改文件，最终他选择不抹除硬盘数据，希望后来者能读到真相。
